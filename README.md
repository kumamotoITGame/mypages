## 🟦 アピールポイント
- プレイヤーの行動によって敵の出現パターンが変化するため、毎回異なる攻略を楽しめます。
- ステージごとに異なるギミックを配置し、単調にならないゲーム体験を実現しました。
- 操作をシンプルにしつつ、タイミングや判断力が求められるゲームデザインを意識しました。
- 初心者でも遊びやすいように、チュートリアルやヒント表示機能を実装しました。

## 💡 工夫した点

本プロジェクトでは以下の点を工夫して設計・実装を行いました。

- 〇〇の処理を最適化し、動作負荷を軽減した
- UIを見やすく設計し、ユーザーの操作性を向上させた
- 難易度調整機能を実装し、プレイヤーのスキルに応じたゲームバランスを実現した

また、オブジェクト指向設計を採用し、各ゲーム要素をクラス単位で責務分割しました。  
共通機能は基底クラスに集約することで、コードの再利用性と拡張性を高めています。  
さらに、イベント駆動による疎結合設計を意識し、クラス間の依存関係を最小限に抑えました。

---

## 💡 苦労した点

敵AIの実装に最も苦戦しました。
当初は条件分岐のみで行動制御を行っていましたが、敵の種類や行動パターンが増えるにつれてコードが複雑化し、管理が困難になりました。
そこで敵の行動を「待機」「追跡」「攻撃」などの状態ごとに分離し、ステートマシンを導入しました。
その結果、

- コードの可読性向上
- 新しい敵の追加が容易になる
- バグ発生箇所の特定がしやすくなる

といった改善を実現できました。
